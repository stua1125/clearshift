## 디자인할 화면: 공유 캘린더 - 주간 뷰

Google Calendar 주간 뷰를 참고하되, 시간축 대신 근무자 카드가 세로로 쌓이는 형태.
핵심: 7개의 날짜 열(column)이 있고, 각 열 안에 해당 날짜에 배정된 근무자들이 세로로 나열된다.
절대로 왼쪽에 이름 열을 두고 가로로 요일을 배치하는 테이블 형태가 아님.

### 화면 구성 (위에서 아래로)

1. 상단 헤더 바
   - 좌측: 뒤로가기 화살표 아이콘
   - 중앙: "전체 근무 일정" (18px Bold)
   - 우측: 월간/주간 전환 아이콘 버튼

2. 주차 네비게이터
   - 좌측: < 이전 주 화살표
   - 중앙: "3월 1주차 (1~7일)" (14px SemiBold)
   - 우측: 다음 주 > 화살표

3. 요일 헤더 행 (7열, 균등 분할)
   - 각 열: 요일 텍스트 (월, 화, 수, 목, 금, 토, 일) + 날짜 숫자
   - 요일: 11px, #8B95A1 (토=파랑, 일=빨강)
   - 날짜 숫자: 16px Bold, #191F28
   - 오늘 날짜: 파란 원(#0064FF) 안에 흰색 숫자
   - 열 사이 구분선: 0.5px #F2F4F6

4. 이벤트 바 영역 (선택적)
   - 해당 주에 걸쳐있는 이벤트를 가로 바로 표시
   - 왼쪽 3px 색상 테두리 + 이벤트 제목
   - 예시: 파란색 바 "단체 정기 교육" (월~수 3일간 걸침)

5. 7열 근무자 카드 그리드 (핵심 영역, 나머지 공간 모두 차지)
   - Google Calendar 주간 뷰처럼 7개의 세로 열
   - 각 열 = 하루
   - 열 사이 구분선: 0.5px #F2F4F6 세로선
   - 각 열 안에 해당 날짜에 배정된 근무자 카드가 위에서 아래로 쌓임
   - 세로 스크롤 가능 (인원이 많을 경우)

   근무자 카드 디자인:
   - 높이: 약 36~40px
   - 좌우 마진: 2px
   - 하단 마진: 3px
   - 왼쪽 3px 두께의 근무타입 색상 사이드바
   - 배경: 근무타입의 연한 배경색 (예: D면 #E8F0FE)
   - 둥근 모서리: 6px
   - 내용 (세로 배치):
     - 1행: 근무타입 약어 (10px Bold, 근무색상) — "D", "N", "OFF"
     - 2행: 근무자 이름 (9px Medium, 근무색상 80% 투명도) — "김철수"

   예시 데이터 (3월 1주차, 월~금 5일):

   월1 열:
   - [#E8F0FE, 좌측 #0064FF 바] D 김철수
   - [#E8F0FE, 좌측 #0064FF 바] D 이영희
   - [#F0EDFF, 좌측 #6C5CE7 바] N 박민준
   - [#FFEBEE, 좌측 #FF3B30 바] E 정수진

   화2 열:
   - [#E8F0FE] D 김철수
   - [#E6F9F3, 좌측 #00B894 바] OFF 이영희
   - [#FFEBEE] E 박민준
   - [#FFEBEE] E 정수진

   수3 열:
   - [#FFEBEE] E 김철수
   - [#E8F0FE] D 이영희
   - [#F0EDFF] N 박민준
   - [#E6F9F3] OFF 정수진

   목4 열:
   - [#E6F9F3] OFF 김철수
   - [#E8F0FE] D 이영희
   - [#F0EDFF] N 박민준
   - [#F0EDFF] N 정수진

   금5 열:
   - [#E6F9F3] OFF 김철수
   - [#FFEBEE] E 이영희
   - [#E6F9F3] OFF 박민준
   - [#F0EDFF] N 정수진

   토6 열: (토요일 — 헤더 날짜 파란색)
   - [#E8F0FE] D 김철수
   - [#E8F0FE] D 이영희
   - [#E8F0FE] D 박민준
   - [#E6F9F3] OFF 정수진

   일7 열: (일요일 — 헤더 날짜 빨간색)
   - [#F0EDFF] N 김철수
   - [#E6F9F3] OFF 이영희
   - [#E6F9F3] OFF 박민준
   - [#E6F9F3] OFF 정수진

6. 하단 네비게이션 바
   - 3탭: 공유 캘린더 (활성, 파란색), 근무 신청, 설정
   - Material 3 스타일, 상단 1px #F2F4F6 구분선

## 절대 하지 말 것

- 왼쪽에 이름 열(Y축)을 두고 가로로 요일을 배치하는 스프레드시트/테이블 형태 금지
- 가로 스크롤 테이블 금지
- 인원별 행(row) 형태 금지

## 반드시 지킬 것

- Google Calendar 주간 뷰처럼 세로 열(column) = 하루
- 각 열 안에 근무자 카드가 위에서 아래로 쌓이는 구조
- Toss 스타일: 순백 배경, 그림자 없음, 1px 보더 기반, 충분한 여백
- 근무타입별 컬러풀한 카드가 한눈에 패턴을 보여주는 것이 핵심

## 참고 이미지 레퍼런스

Google Calendar 주간 뷰를 떠올리되:
- 시간축(hour) 대신 근무자 카드가 세로로 쌓임
- 각 이벤트 블록 = 근무자 카드 (근무타입 색상 + 약어 + 이름)
- 하루에 4~8명의 카드가 세로로 배치됨
